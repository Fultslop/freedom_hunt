# Login & Authentication Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add project-level access control — shared password per project, KV-backed, Worker-issued httpOnly cookies with HMAC-SHA256 signed tokens, rate-limited login, with a full-screen login form and a redesigned TitleBar drill-down menu.

**Architecture:** A Cloudflare KV namespace (`AUTH_STORE`) stores one plaintext password per project. The Worker validates the password, rate-limits attempts by IP, and sets an httpOnly cookie containing an HMAC-SHA256 signed token. The React app calls `GET /auth/me` on mount to learn the current user's identity. A `ProtectedRoute` wraps all project-level routes, checking `activeAuth` against the URL param.

**Key changes from v1:** httpOnly cookies instead of localStorage tokens; no `/auth/required` endpoint; `AuthContext` uses `GET /auth/me` instead of scanning localStorage; rate limiting on `/auth/login`.

**Tech Stack:** Cloudflare Workers (SubtleCrypto API, KV), React 19, React Router v7, Vitest + @testing-library/react, inline styles throughout.

**Spec:** `doc/superpowers/specs/2026-05-01-login-design.md`
